## 2. 우선순위 높은 오픈 이슈

## 2.1 실제 코드 앱 위치

질문:

- 루트 기준 구조를 어디까지 단순화할지
- `src/lib/supabase`, `workers`를 어떤 수준까지 분리할지

현재 결정:

- 새 폴더를 만들지 않고 현재 루트에서 시작
- `apps/web` 전제는 더 이상 사용하지 않음

## 2.2 업로드 저장 UX

질문:

- 브라우저 업로드 즉시 저장할지
- 임시 업로드 후 저장 버튼을 둘지

영향:

- 사용자 혼동 가능성
- API 구조
- 파일 상태 표시

## 2.3 보고서 열람 방식

질문:

- 기존 HTML을 그대로 viewer로 보여줄지
- 새 탭으로 열지
- 일부를 React 화면으로 감쌀지

영향:

- 구현 난이도
- UX 통일성
- 유지보수 비용

## 2.4 인증 1차 범위

현재 결정:

- 개발용 무로그인으로 시작
- 로그인, 세션, 권한 분기 UI는 1차 범위에서 제외

영향:

- 화면 진입 흐름 단순화
- 회사 선택이 핵심 진입 문맥이 됨
- 인증/권한 문서는 2차 이후 범위로 이동

## 2.5 실행 상태 업데이트 방식

질문:

- polling만으로 충분한지
- SSE/WebSocket이 필요한지

현재 권장:

- 1차는 polling
